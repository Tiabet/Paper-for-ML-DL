BLEU, ROUGE, METEOR, F1 이 기존 데이터셋에는 사람 채점과 대체로 상관관계를 보이지만, 답변의 자유도가 높은 설정에서는 안정적인 평가가 어렵다함 → https://aclanthology.org/D19-5817.pdf#:~:text=well,all%20extractive%20QA%20tasks%20depending

현재 지표들이 생성 QA의 복잡성을 제한하며, 보다 추상적이고 다양성 있는 답변을 평가하려면 n-그램 매칭을 넘어서는 새로운 지표가 필요함을 강조 → BERTScore 등의 등장 배경

NarrativeQA (Kocisky et al., 2018, NAACL)처럼 장편 소설 줄거리로부터 질의 응답을 생성하는 과제의 경우, 정답이 한두 단어가 아니라 여러 문장으로 서술되므로 EM으로 정확히 일치시키기가 불가능

→ 이 때문에 NarrativeQA 논문(정답의 표현이 다양할 수 있는 경우)에서는 *“생성된 텍스트 평가를 위해 BLEU-1, BLEU-4, METEOR, ROUGE-L를 사용”*하여 모델 출력을 평가했다고 명시 → https://aclanthology.org/Q18-1023.pdf#:~:text=We%20require%20the%20use%20of,how%20good%20our%20model%20is

  • 각 질문당 2개의 참조 답변 제공, 모델 출력과 두개의 모범 답안 사이의 n-그램 겹침 정도를 위 지표들로 산출
  • ROUGE-L는 정답과 예측 간 가장 긴 공통 부분수열(Longest Common Subsequence)의 길이에 기반한 recall 지표로 핵심 문장 겹침을 측정
  • BLEU-4는 4-그램 정확도로 세부 표현 일치를 평가
  • 자동 평가 외에 랭킹 기반 평가(Mean Reciprocal Rank)도 실시 : 생성된 답변의 품질과 무관하게 모델이 정답 후보들 중 정답을 몇번째로 선호하는지 측정하여 이해력 자체 평가

n-gram 기반은 기존 데이터셋에서는 사람 채점과 대체로 상관관계를 보이지만, 답변이 자유도가 높은 설정(예: 기존 객관식 데이터를 서술형으로 변환)에서는 안정적인 평가가 어렵다고 지적 → https://aclanthology.org/D19-5817.pdf#:~:text=suitable%20for%20all%20extractive%20QA,necessitating%20new%20metrics%20that%20go

RAGAs, ARES → 전제 : 주어진 문헌에 나오지 않는 내용은 틀린것으로 봄

평가자 편향 통제 : 자기선호 편향(self-preference) → 고도화된 프롬프트, 복수평가자(여러 LLM, 앙상블모델), 평가 결과에 대한 설명 점검

통합 평가 활용

  • 지금까지 대부분 RAG평가연구 : Retriver, Generator 성능을 개별 지표로 분리하여 평가
    • 장점 : 어느 부분에 결함이 있는지 파악하기 쉬움
    • 단점 : 최종 답변의 유용성이 실질적으로 중요하기 때문에, 부차적임
